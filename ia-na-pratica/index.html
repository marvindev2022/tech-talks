Hooks de
Automação
"O padrão que depende de lembrar não é um padrão. É uma esperança."

O que são Hooks

Eventos disponíveis

PreToolUse — antes de qualquer tool

PostToolUse — após cada tool

UserPromptSubmit — ao enviar prompt

Stop — ao finalizar resposta

O que você pode fazer

  • Injetar contexto automático no prompt
  • Validar antes de commitar
  • Bloquear comandos perigosos
  • Logar todas as ações da IA
  • Notificar time após cada deploy
  • Otimizar tokens antes de cada chamada

✓ RTK (Rust Token Killer)

Hook que otimiza todos os comandos shell — 60–90% de economia de tokens em tool results.

✓ Guardrails automáticos

Hook que bloqueia git push direto para main e kubectl em produção.

Identifica bottlenecks

Analisa cobertura

Workflow em código

const DIMENSOES = [
  { label: 'security',  prompt: 'Audite vulnerabilidades...' },
  { label: 'perf',      prompt: 'Identifique bottlenecks...' },
  { label: 'coverage',  prompt: 'Analise gaps de cobertura...' },
]

// 3 agentes rodando ao mesmo tempo
const resultados = await parallel(
  DIMENSOES.map(d => () =>
    agent(d.prompt, { label: d.label, schema: FINDINGS_SCHEMA })
  )
)

// cada finding verificado por um adversarial checker
const verificados = await parallel(
  resultados.flat().map(f => () =>
    agent(`Tente refutar: ${f.title}`, { schema: VERDICT_SCHEMA })
  )
)
